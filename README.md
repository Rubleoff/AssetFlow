# AssetFlow

AssetFlow — это полнофункциональное приложение для управления личными финансами с backend на FastAPI, frontend на React/Vite, фоновыми задачами и локальной Docker-средой. Система объединяет ежедневный учёт денежных потоков с активами, вкладами, целями, регулярными платежами, импортом данных, уведомлениями и аналитическими дашбордами.

## Возможности

- Счета и балансы
- Транзакции, переводы, категории и теги
- Бюджеты и уведомления о достижении лимитов
- Активы с ручной оценкой или рыночным отслеживанием цены
- Вклады и регулярные транзакции
- Финансовые цели и планирование пополнений
- Отчёты по сводке, денежному потоку, динамике категорий, мерчантам, аллокации и капиталу
- Сценарное моделирование
- Предпросмотр и применение CSV-импорта
- Ежедневные дайджесты и уведомления об аномалиях
- Административные инструменты для outbox-событий и фоновых задач Celery

## Структура репозитория

```text
.
├── api/                  FastAPI-приложение, SQLAlchemy-модели, Celery-задачи, тесты
├── web/                  React 19 + Vite frontend
├── infra/nginx/          Конфигурация локального reverse proxy
├── scripts/              Экспорт OpenAPI и генерация frontend-клиента
├── docker-compose.yml    Полный локальный стек
└── .env                  Локальные переменные окружения
```

## Архитектура

### Backend

Backend находится в `api/app` и организован вокруг следующих частей:

- `app/main.py`: настройка FastAPI-приложения, CORS, health endpoint, runtime-совместимость схемы БД
- `app/api/routes/*`: REST-эндпоинты для auth, users, accounts, transactions, planning, assets, reports, imports, notifications и admin-инструментов
- `app/models/*`: SQLAlchemy-модели для учёта операций, планирования, портфеля, импорта, уведомлений и снапшотов
- `app/services/*`: доменная логика для аналитики, учёта операций, импорта, сценариев, авторизации, сидирования демо-данных и рыночных данных
- `app/tasks/*`: Celery-приложение и фоновые задачи для регулярных операций, обновления снапшотов и обработки outbox-событий

API использует cookie-based аутентификацию с access и refresh токенами. Backend может автоматически создавать демо-данные при первом запуске и отдаёт OpenAPI-схему для генерации типов на frontend.

### Frontend

Frontend находится в `web/src` и использует:

- TanStack Router для маршрутизации
- TanStack Query для работы с серверным состоянием и инвалидацией кэша
- React Hook Form + Zod для форм и валидации
- Recharts для дашбордов и отчётов
- Сгенерированные OpenAPI-типы в `web/src/lib/generated`

Основные части frontend:

- `app/`: роутер, shell, логика темизации
- `routes/`: страницы приложения
- `components/`: общие UI-компоненты, drawer’ы и overlay
- `lib/`: API-клиент, query-хуки, форматтеры, каталог, уведомления, сгенерированные типы
- `styles/global.css`: дизайн-токены и глобальные стили интерфейса

## Основные сценарии работы

1. Пользователь регистрируется или входит через cookie-based auth.
2. Backend при необходимости создаёт демо-счета, транзакции, бюджеты, активы, цели и уведомления.
3. Пользователь управляет счетами, операциями, бюджетами, активами, вкладами, регулярными платежами и целями.
4. Изменения создают outbox-события.
5. Celery-воркеры обрабатывают события, обновляют аналитические снапшоты и создают дайджесты/уведомления об аномалиях.
6. Frontend использует аналитические эндпоинты для отображения дашбордов, трендов и сценарных прогнозов.

## Локальный запуск

### Вариант 1: Docker Compose

Это самый простой способ запустить весь стек.

```bash
docker compose up --build
```

После запуска приложение будет доступно по адресу `http://localhost:8080`.

Compose поднимает:

- `nginx`
- `web`
- `api`
- `worker`
- `beat`
- `postgres`
- `redis`

### Вариант 2: запуск backend и frontend отдельно

#### Backend

Требования:

- Python 3.9+
- Удобно использовать `uv`, но подойдёт и `venv` + `pip`

Пример настройки:

```bash
cd api
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -e ".[dev]"
uvicorn app.main:app --reload --port 8000
```

По умолчанию backend может работать через SQLite (`sqlite:///./assetflow.db`), если переменные окружения не переопределены.

#### Frontend

Требования:

- Рекомендуется Node.js 22

Пример настройки:

```bash
cd web
npm install
VITE_API_BASE=http://localhost:8000/api npm run dev
```

В `docker-compose.yml` уже заданы разумные локальные значения для большинства из них.

## Генерация API-типов

Frontend-типы генерируются из OpenAPI-схемы backend:

```bash
./scripts/generate_client.sh
```

Скрипт делает следующее:

1. Экспортирует OpenAPI JSON из FastAPI-приложения в `web/src/lib/generated/openapi.json`
2. Перегенерирует TypeScript-описания в `web/src/lib/generated/api-types.d.ts`

Запускайте его после изменения backend-схем или контрактов API.

## Тесты

### Backend

```bash
cd api
pytest
```

Сейчас тесты покрывают smoke-сценарии API, расчёты и поведение уведомлений/outbox-механизма.

### Frontend

```bash
cd web
npm test
```

Проверка типов и сборка:

```bash
cd web
npm run build
```

## Важные детали реализации

- Для поиска и оценки криптоактивов используется CoinGecko, для части рыночных инструментов — Twelve Data.
- Фоновые задачи обновляют снапшоты и генерируют дайджесты/уведомления об аномалиях.
- Пользовательские настройки включают параметры уведомлений и импорта.
- OpenAPI встроен в workflow проекта, поэтому frontend-типы синхронизированы с backend-контрактами.
- Интерфейс поддерживает светлую и тёмную темы.
